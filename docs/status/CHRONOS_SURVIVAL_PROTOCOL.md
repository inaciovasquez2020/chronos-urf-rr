# Chronos Survival Protocol

Status: Repository-governance protocol

## Core Claim

`chronos-urf-rr` is presented as an executable artifact for local-computation, refinement-depth, and status-normalized rigidity claims.

It does not claim that every broader URF or complexity-theoretic theorem is solved.

It requires every Chronos-facing claim to distinguish:

- solved theorem,
- closed executable surface,
- certified frontier,
- conditional result,
- open obstruction.

## Survival Rule

A Chronos artifact survives skeptical review when a reader can determine:

1. the target theorem or certificate,
2. the verified executable surface,
3. the active mathematical frontier,
4. the command that checks the status,
5. whether any theorem-level ingredient remains open.

## Forbidden Claims

`chronos-urf-rr` must not imply that an open theorem is solved because:

- tests pass,
- Lean builds,
- documentation is complete,
- an invariant certificate exists,
- a reduction has been recorded,
- a frontier has been isolated.

## Normal Form

A Chronos component is admissible only when it is one of:

1. a verified theorem or executable certificate,
2. a status-normalization artifact,
3. a certified frontier statement,
4. a conditional result with explicit assumptions,
5. an open obstruction with a named missing ingredient.

## Canonical Public Claim

The durable contribution of `chronos-urf-rr` is the conversion of refinement-depth and local-computation claims into auditable, executable, status-normalized artifacts.

## Continuation Rule

Continue `chronos-urf-rr` only by:

- strengthening verification,
- clarifying theorem status,
- shrinking a frontier,
- proving a registered lemma,
- improving auditability,
- removing ambiguity.

Do not expand Chronos merely by adding new terminology.
